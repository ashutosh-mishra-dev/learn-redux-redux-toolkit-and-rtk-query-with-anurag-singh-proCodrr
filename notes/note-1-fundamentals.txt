    parcel :- Parcel = Bundler

    Parcel ek zero-config bundler hai jo modern JS, CSS, assets ko process karke browser-ready bundle banata hai.

    Agar tum modern JavaScript likhte ho jaise:

    import { createStore } from "redux";
    import "./style.css";
    import logo from "./logo.png";

    Parcel karta kya hai: 
    - ES Modules -Convert karke browser-compatible banata hai
    - Multiple JS files	ko Ek bundle file bana deta hai
    - CSS import :	CSS ko JS se link kar deta
    - Images/fonts :Optimize karke include karta
    - Dev server : Local server chala deta (auto refresh)

Q1. Reducer kya hai ?
   Reducer = State Update Karne Wala Function, reducer hi state update karta hai.
   Reducer pure function hai jo current state aur action lekar NEW STATE return karta hai
  
  ðŸ“ Reducer ke Important Rules:
    Rule 1: Pure Function Hona Chahiye
    
    // âœ… PURE FUNCTION (Best)
      
      const reducer = (state, action) => {
        return { ...state, count: state.count + 1 }; // New object
      };

  // âŒ IMPURE (Never do this!)
    
    const badReducer = (state, action) => {
        state.count++; // Direct mutate âŒ
        return state;
    };
Rule 2: Current State Change Nahi Karna
    // âŒ BAD - Direct mutation
      
      const badReducer = (state, action) => {
          state.todos.push(action.todo); // âŒ Original array change
          return state;
      };

    // âœ… GOOD - New state return karna
      
      const goodReducer = (state, action) => {
          return {
              ...state,
              todos: [...state.todos, action.todo] // âœ… New array
          };
      };

   Q4. Kya action, type, payload predefined hain? 
   Haan aur Nahi! Structure predefined hai, lekin property names nahi
   Redux ki requirement:
    - Action ek OBJECT hona chahiye
    - Usme "type" property honi chahiye
    - "payload" optional hai (convention hai, requirement nahi)

    Q5. what is subscribe?
    Subscribe ek method hai jo aapko batata hai ki Redux store ka state kab change hota hai.
     Yeh ek listener/observer pattern follow karta hai.

     Humme bar-bar console.log ya manual check nahi karna padega, automatically pata chal jayega ki
      state change hua!
